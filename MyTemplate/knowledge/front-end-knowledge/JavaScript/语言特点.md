### JavaScript特点

1. JavaScript 是一种广泛使用的编程语言，主要用于网页开发，使网页具有动态交互功能。

2. JavaScript 是一种解释型语言，代码在运行时由浏览器或运行环境（如 Node.js）逐行解释执行，无需预先编译。

3. JavaScript 通过事件循环机制处理异步操作，使得它可以高效地处理用户输入、网络请求等异步任务。

4. JavaScript 运行在一个单一的主线程上，这意味着同一时刻只能执行一个代码块。这种设计简化了编程模型，避免了多线程环境下的复杂问题，如死锁和资源竞争。但通过事件循环、任务队列、回调函数、Promise 和 async/await 等机制，它可以高效地处理异步操作，从而实现非阻塞的并发执行。
   * 回调函数：回调函数是最基本的异步编程模式之一。当你发起一个异步操作时，你可以传递一个回调函数，在操作完成后执行。
  
   * 事件循环：事件循环是 JavaScript 引擎的核心机制，负责不断检查任务队列并依次执行其中的任务。

   * 任务队列：当异步操作（如 setTimeout 或 AJAX 请求）完成时，它们的结果会被放入任务队列中，等待事件循环来执行。

   * ES6 引入了 Promise 对象   ES8 引入了 async 和 await 关键字

   * JavaScript 中的任务分为两种类型：微任务（microtasks）和宏任务（macrotasks）。它们的区别在于执行时机：
     * 宏任务：包括整体代码块、setTimeout、setInterval、I/O 操作等。每次事件循环只执行一个宏任务。
     * 微任务：包括 Promise 的回调、MutationObserver 等。每次事件循环结束后会清空所有的微任务队列。
