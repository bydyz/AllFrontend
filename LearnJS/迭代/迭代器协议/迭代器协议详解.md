# 什么是迭代器协议？

迭代器协议是JavaScript中的一种协议，定义了如何产生一个序列的值。

一个对象要实现迭代器协议，必须实现一个next()方法，该方法返回一个包含两个属性的对象：

- value: 迭代序列的下一个值
- done: 布尔值，表示序列是否已经结束





# 迭代器协议的应用

实现了迭代器协议的对象的主要用途是 **手动控制迭代过程** ；但 **不能使对象可以在 for...of 等结构中使用** 。





# 实现迭代器协议的步骤:

1. 创建一个对象，该对象包含一个next()方法
2. next()方法必须返回一个对象，包含value和done属性
3. 每次调用next()方法时，返回序列中的下一个值
4. 当没有更多值可返回时，设置done为true
5. **可选**地实现return()和throw()方法用于提前终止迭代
  > 根据 ECMAScript 规范（Iterator 接口）：
  >   * return(value) 用于提前关闭迭代器，其返回值是一个 IteratorResult 对象，通常 done 为 true。
  >   * throw(exception) 用于向迭代器注入异常，返回值也是一个 IteratorResult 对象，done 可以是 true 或 false（例如生成器内部捕获异常后可以继续产生值）。
  >   * 规范没有明确规定调用 return() 或 throw() 后，再调用 next() 必须返回 { done: true }。迭代器的内部状态完全由实现者控制。
  > > 实际中推荐返回 { done: true }
  > > 如果 return() 后仍然产生值，会怎样？
  > >    假设一个迭代器在 return() 后，next() 仍然返回 { value: 1, done: false }：
  > >    * 手动调用 it.next() 会继续得到值，违背用户直觉。
  > >    * 但对于 for...of 循环，一旦调用了 return()（因为 break），循环就不会再使用这个迭代器，因此不会触发额外的 next() 调用。因此，这种“坏”实现在仅用 for...of 时可能不会暴露问题，但代码的可维护性和健壮性会变差。
  > 
  > > throw() 的特殊情况？
  > >    * 在生成器中，如果内部捕获了异常，可以继续 yield 值，此时 throw() 返回 { value: ..., done: false }，且迭代器继续有效。
  > >    * 如果生成器未捕获异常，throw() 会向外抛出异常，迭代器终止，后续 next() 返回 { done: true }。因此，throw() 后迭代器是否完成，取决于实现。
  > 
  > > 最佳实践
  > >    * return() 后：迭代器应进入完成状态，后续所有 next() 返回 { done: true }，并确保资源已清理。
  > >    * throw() 后：根据错误处理逻辑决定是否继续迭代。如果异常未处理，应使迭代器完成；如果处理了且允许继续，则可保持活动。
  >
  > > ```javascript
  > > function createRange(start, end) {
  > >   let current = start;
  > >   let isClosed = false;   // 标记是否已提前关闭
  > >   return {
  > >     next() {
  > >       if (isClosed) return { done: true };
  > >       if (current < end) {
  > >         return { value: current++, done: false };
  > >       }
  > >       return { done: true };
  > >     },
  > >     return() {
  > >       if (!isClosed) {
  > >         console.log('清理资源...');
  > >         isClosed = true;
  > >         current = end; // 确保后续 next 直接结束
  > >       }
  > >       return { done: true };
  > >     },
  > >     [Symbol.iterator]() { return this; }
  > >   };
  > > }
  > > ```



# 总结

迭代器协议是针对对象而言的。在JavaScript中，任何对象 **只要实现了next()方法，就满足了迭代器协议** 。

**虽然任何对象都可以实现next()方法，但只有具有 `[Symbol.iterator]` 方法的对象才是可迭代对象** 。

* 迭代器: 任何实现了next()方法的对象
* 可迭代对象: 具有 `[Symbol.iterator]` 方法的对象，该方法返回一个迭代器
