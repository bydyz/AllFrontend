# 特点

1. 没有副作用（对整个项目没有任何其他影响，只为得到一个值，因此将函数换为 得到的值 亦可）
2. 相同的输入，永远会得到相同的输出（即函数的返回值只依赖于传入的参数，与任何外部状态无关）。
3. 它的存在只是为了返回一个值



# 优点

1. 测试纯函数非常简单：只需给定输入，断言输出即可。不需要模拟环境、设置全局状态或清理副作用。单元测试更容易编写和维护。
2. 由于相同的输入总是返回相同的输出，纯函数的结果可以被缓存起来。例如，可以用一个 Map 存储参数和结果的对应关系，下次调用时直接返回缓存值，提升性能。
3. 纯函数之间没有共享状态，因此可以安全地在多线程或分布式环境中并行执行，无需担心竞态条件或锁机制。
4. 纯函数的代码更容易理解和推理，因为函数的行为是自包含的，不依赖外部上下文。修改一个纯函数不会意外影响程序的其他部分，降低了耦合度。
5. 纯函数可以像积木一样组合成更复杂的功能，因为它们的输入输出关系清晰，且没有副作用干扰组合过程。
