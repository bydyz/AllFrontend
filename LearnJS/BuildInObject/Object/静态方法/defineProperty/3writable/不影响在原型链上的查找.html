writable: false 与原型链属性查找

详解 writable: false 如何影响属性赋值但不影响原型链上的属性查找

属性描述符与 writable 属性

在JavaScript中，每个属性都有一个属性描述符，用于定义属性的特性。其中writable属性决定是否可以修改属性的值。

属性描述符包含以下属性：

  • value：属性的值
  • writable：是否可写（默认为false）
  • enumerable：是否可枚举
  • configurable：是否可配置
// 使用 Object.defineProperty 定义属性
const obj = {};
Object.defineProperty(obj, 'readOnlyProp', {
  value: '初始值',
  writable: false, // 不可写
  enumerable: true,
  configurable: false
});

// 尝试修改只读属性
obj.readOnlyProp = '新值'; // 静默失败或TypeError（严格模式）
console.log(obj.readOnlyProp); // '初始值'

writable: false 的含义

writable: true

属性值可以被修改

const obj = {};
Object.defineProperty(obj, 'prop', {
  value: '可修改',
  writable: true
});

obj.prop = '新值'; // 成功
console.log(obj.prop); // '新值'

writable: false

属性值不能被修改

const obj = {};
Object.defineProperty(obj, 'prop', {
  value: '不可修改',
  writable: false
});

obj.prop = '新值'; // 静默失败
console.log(obj.prop); // '不可修改'

注意：在严格模式下，尝试修改writable: false的属性会抛出TypeError。

原型链属性查找机制

JavaScript对象有一个指向另一个对象（原型）的内部链接。当访问对象属性时，如果对象自身没有该属性，JavaScript会沿着原型链向上查找。

1
在对象自身查找属性
2
如果对象自身没有该属性，查找其原型
3
如果原型上也没有，继续查找原型的原型
4
直到找到属性或到达原型链末端（null）
// 原型链示例
const parent = {
  parentProp: '父属性'
};

const child = Object.create(parent);
child.childProp = '子属性';

// 属性查找
console.log(child.childProp); // '子属性'（自身属性）
console.log(child.parentProp); // '父属性'（原型链上的属性）

writable: false 不影响原型链属性查找

关键点：writable: false只影响属性赋值操作，不影响属性查找过程。

示例：原型上的只读属性

// 创建原型对象，包含一个只读属性
const prototypeObj = {};
Object.defineProperty(prototypeObj, 'readOnly', {
  value: '原型上的只读值',
  writable: false,
  enumerable: true
});

// 创建子对象，继承自原型
const childObj = Object.create(prototypeObj);

// 1. 属性查找 - 可以正常访问原型上的只读属性
console.log(childObj.readOnly); // '原型上的只读值'

// 2. 属性赋值 - 尝试修改只读属性
childObj.readOnly = '新值'; // 静默失败（非严格模式）

// 3. 属性查找 - 仍然访问原型上的值
console.log(childObj.readOnly); // '原型上的只读值'

// 4. 创建同名自有属性（遮蔽原型属性）
Object.defineProperty(childObj, 'readOnly', {
  value: '自有属性值',
  writable: true,
  enumerable: true
});

// 5. 现在访问的是自有属性
console.log(childObj.readOnly); // '自有属性值'

关键理解：

  • writable: false阻止的是赋值操作，不是读取操作
  • 属性查找过程完全不受writable标志的影响
  • 即使原型属性是只读的，仍然可以创建同名的自有属性来"遮蔽"它

属性设置与遮蔽

当给对象属性赋值时，JavaScript会执行一系列步骤来决定如何处理这个赋值操作。

1
如果对象自身有该属性且可写，直接修改值
2
如果对象自身有该属性但不可写，赋值失败（严格模式抛出错误）
3
如果对象自身没有该属性，继续在原型链上查找
4
如果原型链上的属性可写，在原始对象上创建新属性（遮蔽）
5
如果原型链上的属性不可写，赋值失败（严格模式抛出错误）
6
如果原型链上没有该属性，在原始对象上创建新属性

属性赋值规则示例

// 情况1：原型属性可写
const proto1 = { writableProp: '可写' };
const obj1 = Object.create(proto1);
obj1.writableProp = '新值'; // 成功 - 在obj1上创建新属性
console.log(obj1.writableProp); // '新值'
console.log(proto1.writableProp); // '可写'（未被修改）

// 情况2：原型属性不可写
const proto2 = {};
Object.defineProperty(proto2, 'readOnlyProp', {
  value: '不可写',
  writable: false
});
const obj2 = Object.create(proto2);
obj2.readOnlyProp = '新值'; // 失败（非严格模式静默失败）
console.log(obj2.readOnlyProp); // '不可写'（仍然访问原型属性）

实际应用场景

防止意外修改

使用writable: false可以防止重要属性被意外修改。

function createConstants() {
  const constants = {};
  Object.defineProperty(constants, 'PI', {
    value: 3.14159,
    writable: false
  });
  return constants;
}

const math = createConstants();
math.PI = 3; // 静默失败
console.log(math.PI); // 3.14159

保护原型属性

保护原型上的重要方法不被覆盖。

function SafeArray() {
  // 调用父构造函数
  Array.call(this);
}

// 继承Array原型
SafeArray.prototype = Object.create(Array.prototype);

// 保护重要方法
Object.defineProperty(SafeArray.prototype, 'push', {
  value: Array.prototype.push,
  writable: false,
  configurable: false
});

const safeArr = new SafeArray();
safeArr.push = function() { /* 恶意代码 */ }; // 失败
