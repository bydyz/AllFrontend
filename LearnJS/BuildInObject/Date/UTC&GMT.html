UTC与GMT的区别详解

协调世界时与格林尼治标准时间的比较

什么是GMT？

GMT（Greenwich Mean Time，格林尼治标准时间）是指位于英国伦敦郊区的皇家格林尼治天文台的标准时间。

GMT基于地球的自转，以太阳穿越格林尼治子午线的时间作为中午12点。它是19世纪中期为了铁路和航海需要而确立的时间标准。

// JavaScript中获取GMT时间 const date = new Date(); console.log(date.toGMTString()); // 示例输出: "Wed, 04 Oct 2023 12:34:56 GMT"

注意： toGMTString()方法已弃用，建议使用toUTCString()代替。

什么是UTC？

UTC（Coordinated Universal Time，协调世界时）是以原子时秒长为基础，在时刻上尽量接近于世界时的时间计量系统。

UTC通过添加或删除闰秒来保持与UT1（基于地球自转的世界时）的接近，误差不超过0.9秒。

// JavaScript中获取UTC时间 const date = new Date(); console.log(date.toUTCString()); // 示例输出: "Wed, 04 Oct 2023 12:34:56 GMT" // 获取UTC时间组件 console.log(date.getUTCHours()); // UTC小时 console.log(date.getUTCMinutes()); // UTC分钟

主要区别

UTC和GMT的主要区别在于：

  • 基础不同：GMT基于地球自转，UTC基于原子时
  • 精度不同：UTC更精确，GMT受地球自转速度变化影响
  • 闰秒处理：UTC通过闰秒调整，GMT没有闰秒概念
  • 现代应用：UTC是国际标准，GMT更多是历史术语

实际使用： 在日常生活中，UTC和GMT通常可以互换使用，因为它们的时区偏移量相同(UTC+0)。

时间标准发展历史

1884年

国际子午线会议将格林尼治子午线定为本初子午线，GMT成为世界时间标准。

1928年

国际天文联合会首次提出"世界时(UT)"的概念，用于区分基于格林尼治平太阳时的GMT。

1960年

国际无线电咨询委员会提出了"协调世界时"的概念，试图协调原子时和世界时。

1963年

UTC时间系统正式建立，成为新的国际时间标准。

1972年

引入闰秒制度，使UTC与UT1的偏差保持在0.9秒以内。

现今

UTC已成为国际标准时间，广泛应用于科学、技术、航空和互联网等领域。

JavaScript中的UTC和GMT

// 创建一个日期对象 const date = new Date(); // 获取UTC/GMT时间表示 console.log(date.toUTCString()); // UTC表示 console.log(date.toGMTString()); // GMT表示 (已弃用) // 获取时间组件 console.log("UTC小时: " + date.getUTCHours()); console.log("UTC分钟: " + date.getUTCMinutes()); console.log("UTC秒: " + date.getUTCSeconds()); // 设置UTC时间 date.setUTCHours(12); date.setUTCMinutes(0); date.setUTCSeconds(0); console.log("设置后的UTC时间: " + date.toUTCString());
