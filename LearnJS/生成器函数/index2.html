Generator函数return行为详解

理解Generator函数中return语句后继续调用next()的行为

Generator函数基础

Generator函数是ES6引入的特殊函数，可以通过yield关键字暂停执行，并通过next()方法恢复执行。

// 基本的Generator函数 function* simpleGenerator() { yield '第一步'; yield '第二步'; return '完成'; } const gen = simpleGenerator(); console.log(gen.next()); // { value: '第一步', done: false } console.log(gen.next()); // { value: '第二步', done: false } console.log(gen.next()); // { value: '完成', done: true }

return语句的作用

在Generator函数中，return语句有两个作用：

  • 返回指定的值作为最后一次next()调用的结果
  • 将Generator的状态设置为已完成(done: true)
function* exampleGenerator() { yield '第一个值'; return '最终返回值'; yield '这行不会执行'; // 不会执行 } const gen = exampleGenerator(); console.log(gen.next()); // { value: '第一个值', done: false } console.log(gen.next()); // { value: '最终返回值', done: true } console.log(gen.next()); // { value: undefined, done: true }

return后的next()调用

当Generator函数执行到return语句后，所有后续的next()调用都会返回相同的值：

function* demoGenerator() { yield 1; yield 2; return 3; } const gen = demoGenerator(); console.log(gen.next()); // { value: 1, done: false } console.log(gen.next()); // { value: 2, done: false } console.log(gen.next()); // { value: 3, done: true } console.log(gen.next()); // { value: undefined, done: true } console.log(gen.next()); // { value: undefined, done: true }

注意： 虽然返回值是undefined，但done属性始终保持为true。

makeRangeIterator函数分析

function* makeRangeIterator(start = 0, end = Infinity, step = 1) { let iterationCount = 0; for (let i = start; i < end; i += step) { iterationCount++; yield i; } return iterationCount; }

函数执行过程:

初始化iterationCount为0
进入for循环，从start开始，到end结束（不包括end）
每次循环iterationCount增加1，并yield当前值i
循环结束后，返回iterationCount（迭代次数）
此后所有next()调用都返回{ value: undefined, done: true }
