# `/*#__PURE__*/` 特征

`/*#__PURE__*/` 有非常重要的作用，尤其是在现代前端构建（如使用 Webpack、Vite、Rollup 等打包工具）时。

`/*#__PURE__*/` 是一个 “纯函数”提示注释，它告诉打包工具：“这个函数调用是纯的（没有副作用），如果它的返回值没有被使用，可以安全地删除它。”

这是为了实现 “tree-shaking”（摇树优化），减少最终打包体积。



# 作用

加上 /*#__PURE__*/ 后，就相当于告诉打包工具：“我保证 defineComponent(...) 这个调用是纯的，它没有副作用。如果 RouterLinkImpl 这个变量最终没有被使用，请大胆删掉它！”

这样就能实现 tree-shaking：未使用的组件不会被打包进最终产物。



# 🌰 举个例子

假设你有一个库：

```javascript
// components.js
export const ComponentA = /*#__PURE__*/ defineComponent({ /* ... */ });
export const ComponentB = /*#__PURE__*/ defineComponent({ /* ... */ });
export const ComponentC = /*#__PURE__*/ defineComponent({ /* ... */ });
```

如果你的项目只用了 ComponentA，打包工具会：
1. 分析依赖
2. 发现 ComponentB 和 ComponentC 未被引用
3. 由于它们是 /*#__PURE__*/ 标记的，可以安全删除
4. 最终打包文件只包含 ComponentA
✅ 减小了包体积。



# 🔍 实际效果（Vite + Rollup / Webpack）

在生产构建时，打包工具会进行 DCE（Dead Code Elimination，死代码消除）。

如果没有 `/*#__PURE__*/`：
```javascript
var ComponentB = defineComponent({ /* ... */ }); // 即使没用，也可能保留
```

有 `/*#__PURE__*/`：
```javascript
// components.js
export const ComponentA = /*#__PURE__*/ defineComponent({ /* ... */ });
export const ComponentB = /*#__PURE__*/ defineComponent({ /* ... */ });
export const ComponentC = /*#__PURE__*/ defineComponent({ /* ... */ });
```



# 🚫 注意事项

* `/*#__PURE__*/` 只是提示，不是强制。
* 如果函数实际上有副作用，你却标记为 pure，会导致 bug。
  * 比如：`/*#__PURE__*/` console.log('hello') ❌ 错误！console.log 有副作用（打印日志），不能标记为 pure。
* 它只对**立即执行的函数调用**有效。



# ✅ Vue 中为什么广泛使用？

Vue 3 的源码中大量使用 `/*#__PURE__*/`，因为：
* Vue 是一个库，必须支持 tree-shaking。
* 很多组件（如 RouterLink、Transition）可能被用户不用。
* 标记为 pure，可以让未使用的组件被摇掉，减少用户打包体积。
