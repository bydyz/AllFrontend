`package.json` 是 **Node.js** 项目 的核心配置文件，几乎所有基于 JavaScript 或 Node.js 的项目（前端、后端、工具库等）都会用到它。

***

# 🧑‍💻 谁要用 package.json？

1. 包管理工具（如 npm、yarn、pnpm）
    * 它们通过 package.json 知道项目需要安装哪些依赖（dependencies 和 devDependencies），以及如何执行脚本（scripts）。<br><br>

2. 开发者
    * 通过 package.json 快速了解项目信息（名称、版本、描述、作者等）。
    * 使用 scripts 中定义的命令（如 npm start、npm test）来运行开发、测试、构建任务。
    * 手动编辑文件来添加或更新依赖，或者配置其他工具（如 ESLint、Babel 等）。
     
3. 构建工具和打包工具（如 Webpack、Vite、Rollup）
    * 它们会读取 package.json 中的 main、module、browser 等字段来确定入口文件，或者通过 exports 字段来控制模块的导出方式。<br><br>

4. 部署环境和 CI/CD 系统
    * 在生产环境或持续集成流程中，会自动运行 npm install 来安装依赖，依赖列表正是来自 package.json（配合 package-lock.json 保证版本一致）。<br><br>

5. npm 仓库本身
    * 当你要把项目发布为 npm 包时，package.json 中的 name 和 version 决定了包的名称和版本号，main 或 exports 指定了包的入口，files 控制哪些文件会被发布。

***

# 📝 package.json 的主要作用

1. 管理项目依赖
    * dependencies：项目运行所必需的包（如 React、Express）。
    * devDependencies：仅在开发时需要，不会在生产环境安装的包（如测试框架、构建工具）。
    * 包管理工具通过这些列表来下载、更新、删除依赖。<br><br>

2. 定义项目元数据
    * name：项目名称（如果是发布到 npm，则必须是唯一的）。
    * version：遵循语义化版本（SemVer），如 1.0.0。
    * description：简短描述，帮助别人了解项目用途。
    * author、license、repository 等：提供作者、许可证和源码仓库信息。<br><br>

3. 提供可执行脚本
    * "scripts" 字段可以自定义命令
      ```json
      "scripts": {
        "start": "node server.js",
        "build": "webpack --mode production",
        "test": "jest"
      }
      ```
      然后通过 npm run start 等来执行，简化重复操作。<br><br>

4. 指定项目入口与模块规范
    * main：最常见的入口文件（如 index.js），供 CommonJS 环境使用。
    * module：指定 ES Module 入口，方便打包工具进行 tree shaking。
    * browser：针对浏览器环境的入口。
    * exports：更精细地控制子路径导出，限制外部访问内部文件。<br><br>

5. 配置工程工具
    * "type": "module" 表示项目使用 ES Module 规范。
    * "engines" 指定 Node.js 或 npm 的版本要求。
    * "browserslist" 配置目标浏览器，被 Babel、Autoprefixer 等工具读取。
    * "jest"、"eslintConfig"、"prettier" 等也可以直接嵌入，不过现在更常见的是使用独立的配置文件。

6. 控制包发布
    * private：设为 true 可防止意外将项目发布到 npm 仓库。
    * files：数组，指定哪些文件会被包含在发布的包中（例如 ["dist", "lib"]）。
    * publishConfig：设置发布时的注册表地址或访问权限。
