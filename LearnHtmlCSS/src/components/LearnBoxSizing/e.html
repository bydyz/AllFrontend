✅ 现代最佳实践

推荐使用： * { box-sizing: border-box; }
border-box
设置: 200×100
实际: 200×100 

/* CSS Reset 最佳实践 */
*,
*::before,
*::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
}

/* 这样设置后： */
.element {
    width: 300px;      /* 总宽度就是 300px */
    padding: 20px;     /* 包含在 300px 内 */
    border: 2px solid; /* 包含在 300px 内 */
}

❌ 传统做法（不推荐）

问题： 需要手动计算内容宽度，布局容易出错
content-box
设置: 200×100
实际: 260×160 

/* 传统做法的问题 */
.element {
    width: 300px;      /* 只是内容宽度 */
    padding: 20px;     /* +40px = 340px */
    border: 2px solid; /* +4px = 344px */
    /* 实际占用 344px，不是预期的 300px */
}

/* 需要手动计算 */
.element {
    width: 256px;      /* 300 - 20×2 - 2×2 = 256 */
    padding: 20px;
    border: 2px solid;
    /* 现在实际是 300px，但计算复杂 */
}

📊 两种模式的对比

特性 border-box content-box
布局计算 简单直观 复杂易错
响应式设计 友好 困难
维护性 高 低
现代框架 默认使用 很少使用
行业标准： Bootstrap、Tailwind CSS、Material-UI 等现代框架都使用 border-box
